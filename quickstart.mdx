Get your LoL Tracker Backend up and running in just a few minutes. This guide will walk you through cloning the repository, setting up your environment, and making your first API calls.

## Prerequisites

Before you begin, make sure you have:

- **Node.js** (v14 or higher) installed on your machine
- **npm** package manager
- A **Riot Games Developer API Key** (we'll show you how to get one)

<Tip>
If you don't have Node.js installed, download it from [nodejs.org](https://nodejs.org/)
</Tip>

## Setup Process

<Steps>
  <Step title="Clone the Repository">
    First, clone the LoL Tracker Backend repository to your local machine:

    ```bash
    git clone https://github.com/ValentinSetti/tracker-lol-backend.git
    cd tracker-lol-backend
    ```
  </Step>

  <Step title="Install Dependencies">
    Install all required npm packages:

    ```bash
    npm install
    ```

    This will install:
    - **express** (v5.2.1) - Web server framework
    - **axios** (v1.13.5) - HTTP client for Riot API requests
    - **cors** (v2.8.6) - Enable cross-origin requests
    - **dotenv** (v17.3.1) - Environment variable management
  </Step>

  <Step title="Get Your Riot API Key">
    To access Riot Games data, you need a developer API key:

    1. Visit [developer.riotgames.com](https://developer.riotgames.com/)
    2. Sign in with your Riot Games account
    3. Navigate to your dashboard
    4. Copy your **Development API Key**

    <Warning>
    Development API keys expire after 24 hours and have rate limits. For production use, you'll need to apply for a production key.
    </Warning>
  </Step>

  <Step title="Configure Environment Variables">
    Create a `.env` file in the root directory of the project:

    ```bash
    touch .env
    ```

    Add your Riot API key to the `.env` file:

    ```bash .env
    RIOT_API_KEY=your_riot_api_key_here
    ```

    <Note>
    The `.env` file is gitignored by default to keep your API key secure. Never commit this file to version control.
    </Note>
  </Step>

  <Step title="Start the Server">
    Launch the development server:

    ```bash
    node index.js
    ```

    You should see:
    ```
    Servidor corriendo en http://localhost:3000
    ```

    The API is now running on port 3000!
  </Step>

  <Step title="Verify the Server">
    Test that the server is running by visiting the root endpoint:

    ```bash
    curl http://localhost:3000
    ```

    Expected response:
    ```
    ¡El servidor del LoL Tracker está vivo, Guillote!
    ```
  </Step>
</Steps>

## Making Your First API Calls

Now that your server is running, let's test the two main endpoints with real API calls.

### Get Player Information

Retrieve a player's rank, level, and profile information:

<CodeGroup>

```bash cURL
curl http://localhost:3000/api/jugador/HideOnBush/KR1
```

```javascript JavaScript (fetch)
fetch('http://localhost:3000/api/jugador/HideOnBush/KR1')
  .then(response => response.json())
  .then(data => console.log(data));
```

```javascript JavaScript (axios)
const axios = require('axios');

axios.get('http://localhost:3000/api/jugador/HideOnBush/KR1')
  .then(response => console.log(response.data));
```

</CodeGroup>

**Example Response:**

```json
{
  "userTier": "CHALLENGER",
  "userRank": "I",
  "userLps": 1247,
  "userWins": 145,
  "userLosses": 98,
  "userLevel": 387,
  "userPic": 4568,
  "userId": "abc123-def456-ghi789..."
}
```

<Tip>
Replace `HideOnBush/KR1` with any valid Riot ID in the format `gameName/tagLine`
</Tip>

### Get Match History

Retrieve a player's recent match history with pagination:

<CodeGroup>

```bash cURL (Default)
curl http://localhost:3000/api/historial/HideOnBush/KR1
```

```bash cURL (With Pagination)
curl "http://localhost:3000/api/historial/HideOnBush/KR1?inicio=0&cantidad=5"
```

```javascript JavaScript (fetch)
fetch('http://localhost:3000/api/historial/HideOnBush/KR1?inicio=0&cantidad=5')
  .then(response => response.json())
  .then(data => console.log(data));
```

</CodeGroup>

**Query Parameters:**

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `inicio` | number | 0 | Starting index for pagination |
| `cantidad` | number | 10 | Number of matches to retrieve |

## Response Field Reference

### Player Endpoint (`/api/jugador/:nombre/:tag`)

| Field | Type | Description |
|-------|------|-------------|
| `userTier` | string | Rank tier (IRON, BRONZE, SILVER, GOLD, PLATINUM, DIAMOND, MASTER, GRANDMASTER, CHALLENGER, or UNRANKED) |
| `userRank` | string | Division within tier (I, II, III, IV) - empty for UNRANKED |
| `userLps` | number | League Points (LP) in current division |
| `userWins` | number | Total ranked wins in current season |
| `userLosses` | number | Total ranked losses in current season |
| `userLevel` | number | Summoner level |
| `userPic` | number | Profile icon ID |
| `userId` | string | Player's unique PUUID |

### Match History Endpoint (`/api/historial/:nombre/:tag`)

| Field | Type | Description |
|-------|------|-------------|
| `duracion` | string | Match duration in MM:SS format |
| `cola` | string | Queue type (e.g., "Clasificatoria Solo/Dúo", "ARAM") |
| `campeon` | string | Champion played by the queried player |
| `estado` | boolean | Whether the queried player won (true) or lost (false) |
| `k` | number | Kills by the queried player |
| `d` | number | Deaths by the queried player |
| `a` | number | Assists by the queried player |
| `jugadoresPartida` | array | Array of all 10 players in the match |
| `killsRed` | number | Total kills by red team |
| `deathRed` | number | Total deaths by red team |
| `assistsRed` | number | Total assists by red team |
| `killsBlue` | number | Total kills by blue team |
| `deathsBlue` | number | Total deaths by blue team |
| `assistsBlue` | number | Total assists by blue team |

## Supported Queue Types

The API translates Riot's queue IDs to human-readable names:

| Queue ID | Display Name |
|----------|--------------|
| 420 | Clasificatoria Solo/Dúo |
| 400 | Normal Reclutamiento |
| 430 | Normal Selección Oculta |
| 440 | Clasificatoria Flexible |
| 450 | ARAM |
| 700 | Clash |
| Other | Otro Modo |

## Troubleshooting

<AccordionGroup>
  <Accordion title="Error: 403 Forbidden from Riot API">
    This usually means your API key is invalid or expired. Development API keys expire after 24 hours.
    
    **Solution:** Get a fresh API key from [developer.riotgames.com](https://developer.riotgames.com/) and update your `.env` file.
  </Accordion>

  <Accordion title="Error: 404 Player Not Found">
    The player name and tag combination doesn't exist.
    
    **Solution:** Verify the exact spelling and tag. Riot IDs are case-sensitive for the game name but tags are always uppercase.
  </Accordion>

  <Accordion title="Error: 429 Rate Limit Exceeded">
    You've made too many requests. Development API keys have strict rate limits:
    - 20 requests per second
    - 100 requests per 2 minutes
    
    **Solution:** Implement request throttling or apply for a production API key with higher limits.
  </Accordion>

  <Accordion title="Server won't start / Port 3000 in use">
    Another application is using port 3000.
    
    **Solution:** Either stop the other application or change the port in `index.js` by modifying the `PORT` constant (line 7).
  </Accordion>
</AccordionGroup>
